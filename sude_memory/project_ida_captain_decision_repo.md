**CRITICAL clarification from user (2026-07-14) — what actually drives the boat during the video:** the capability video only needs to prove İDA moves stably on water. For this specific deliverable, **ArduPilot's own AUTO mode drives the boat directly** (waypoints uploaded via QGC, FC firmware follows them natively) — **neither `ida_topics`'s `decision_node`/`control_node` NOR `girdap_decision`'s `planning_node`/`fsm_node`/`mavros_bridge_node` are in the loop.** This matches `telemetry_node`'s `setpoint_source: "fc"` mode (reads thrust from `/mavros/rc/out` PWM, ignores MPPI thrust entirely — see the B2 fix in [[project_ida_captain_decision_repo]]'s KOD_DEGISIKLIKLERI.txt section). Custom decision/control nodes only matter for **future actual competition parkurs** (autonomous navigation), not this video.

**Node-by-node comparison done 2026-07-14** (ida_topics vs girdap_decision, both under `~/ros2_ws/son_kod/`, see [[reference_ida_file_locations]] for the copy at `~/Masaüstü/ida_son_yazılım/`): neither codebase alone is safety-complete for the *future* competition-driving case. Key findings: girdap_decision has **zero real hardware sensor drivers** (only `mock_sensors.py`, no `/diagnostics` health monitoring) — depends entirely on `ida_topics`-style drivers or stock ROS packages; girdap_decision has **zero RC-hardware kill-switch** (`ida_topics`'s `control_node.py` channel-8 RC kill has no equivalent — grep confirms zero `rc/in` usage under `son_kod/karar/`), but girdap_decision's `mavros_bridge_node.py` has unexpected-disarm detection + KILL-latch that `ida_topics` lacks. girdap_decision's `telemetry_node.py` and `local_map_node.py` are strictly better (fsync, real occupancy data vs ida_topics' fabricated free-circle around the vehicle). girdap_decision's LiDAR clustering is real and hardware-tested but its YOLO perception is mock-only (no trained model wired up) — `ida_topics` has the real trained `best.pt`. Bugs found: `ida_topics/perception_node.py` never publishes `all_detections` (dead code / silent data loss), `ida_topics/control_node.py` blocks an executor thread with `time.sleep(1.0)` inside a subscription callback.
